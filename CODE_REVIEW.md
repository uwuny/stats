# Аудит кода

Дата проверки: 2026-05-11

## Обнаруженные неточности и ошибки

1. **Жёстко зашит год `2025` при разборе даты**
   - В `parseDate` используется `new Date(2025, m-1, d)`, из-за чего сортировка дат становится некорректной при данных из других лет.
   - Файлы: `index.html`, `tables.js`.

2. **Дублирование/рассинхронизация логики между `index.html` и `tables.js`**
   - Практически одинаковая логика построения таблицы присутствует в двух местах.
   - При этом в `index.html` уже есть обновления (например, `DATA_FILES`, детализированный вывод `assist`), а `tables.js` содержит старую версию.
   - Это риск расхождения поведения и ошибок при дальнейших правках.

3. **`tables.js` вероятно не используется страницей**
   - В `index.html` нет подключения `<script src="tables.js">`, весь JS уже встроен в страницу.
   - В результате правки в `tables.js` не влияют на приложение (если файл не подключается где-то ещё).

4. **Несогласованность пути к данным**
   - В `tables.js` загрузка идёт из `battle_data.json`, а в `index.html` — из списка `data/*.json`.
   - Это создаёт разные источники данных в зависимости от того, какой скрипт реально используется.

5. **Дублирующиеся CSS-блоки для `.bottom-row` и `.bottom-row img`**
   - Правила объявлены повторно, что усложняет поддержку и повышает риск случайных визуальных конфликтов.

6. **Нет обработки ошибок при загрузке JSON**
   - `fetch(...).then(r => r.json())` используется без проверки `r.ok` и без `.catch(...)`.
   - При недоступности файлов интерфейс молча ломается без понятного сообщения.

7. **Риск XSS/инъекций через `innerHTML`**
   - Данные (`name`, `tank`, `map`) вставляются в HTML строкой без экранирования.
   - Если источник данных будет скомпрометирован, возможно выполнение произвольного HTML/JS.

8. **Неаккуратность нейминга favicon-файлов**
   - В репозитории есть `Fawicon.png` (опечатка в названии), а в HTML используется `favicon.svg`.
   - Это не критичная runtime-ошибка, но признак технического долга и возможной путаницы.
